# Governance

**How we manage and improve delivery** — the disciplines that ensure work is planned, tracked, and continuously improved.

Governance disciplines provide the management framework within which Engineering, Data, and Product operate. They answer questions about timelines, budgets, scope, risk, and process improvement.

| Discipline | Core question | Package | Deep dives |
|-----------|---------------|---------|------------|
| [**Project Management**](pm/README.md) | Are we delivering on time, on budget, within scope, with acceptable risk? | Process groups, governance, knowledge areas, metrics, approaches (PMBOK, PRINCE2, Six Sigma) | [Approaches](pm/approaches/README.md) · [Governance](pm/governance/README.md) |
| **Lean** *(planned)* | How do we continuously improve and eliminate waste? | Toyota Production System principles, value stream mapping, waste elimination, kaizen | |

## Relationship to other families

| Family | How Governance relates |
|--------|------------------------|
| [**Engineering**](../engineering/README.md) | PM governs engineering delivery — sprint planning, release management, risk tracking. Lean targets engineering process improvement. |
| [**Data**](../data/README.md) | Data projects follow the same delivery governance. PM tracks data pipeline delivery alongside application features. |
| [**Product**](../product/README.md) | Product priorities (from BA, Marketing) flow into PM's backlog and roadmap management. PM ensures product decisions are delivered within constraints. |
| [**Security**](../security/README.md) | PM tracks security work items alongside features. Security incidents trigger PM's risk management processes. |
| [**Compliance**](../compliance/README.md) | Compliance deadlines and audit schedules are managed through PM governance. Lean process improvements must maintain compliance controls. |
