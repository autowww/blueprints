Ceremonies — SDD I/O (C1–C6)

Each foundation intent has preconditions, inputs, outputs, postconditions, and a filled example. Map calendar events via methodology-bridge.md. Canonical: ceremonies-sdd.md. Template: CEREMONY-INTENT.template.md.

C1 — Align on intent SDD-C1-ALIGN

Shared goals, scope, constraints for a horizon before commitment (C2).

Preconditions

IDArtifactOwnerReady when
PRE-C1-1Stakeholder themes / problemsDemand≥3 prioritized themes or ranked backlog
PRE-C1-2Constraints (time, compliance, tech)Steer / TechMust / must-not stated
PRE-C1-3Horizon labelDemandNamed (e.g. M2, Sprint 12)

Inputs (examples)

IDNameExample snippet
IN-C1-1Product goal“Default mobile capture for field notes in EU SMB.”
IN-C1-2Candidate scopeM2E1, M2E3, M2E5
IN-C1-3Dependency assumptions“OCR GA depends on vendor API v3 by Mar 1.”

Outputs (examples)

IDNameConsumerExample
OUT-C1-1Aligned goal statementC2, C4“M2 delivers offline capture + sync conflict UI; no new integrations.”
OUT-C1-2Non-goalsAll“No desktop editor; SSO Google only.”
OUT-C1-3Open questions / spikesBuildSPIKE-1: quota in airplane mode — @alex, before planning
Filled example — M2 kickoff: Goal: offline + conflict UX; non-goals: team workspaces, billing; question: binary attachment merge — PO + tech spike by Jan 15.

C2 — Commit / select work SDD-C2-COMMIT

Explicit choice of work entering the next execution slice with capacity visible.

Preconditions

  • C1 goal merged; top items meet Definition of Ready; capacity known.

Outputs (examples)

IDNameExample
OUT-C2-1Committed setM2E1S1, M2E1S2 only
OUT-C2-2Forecast / sprint goal“Finish offline MVP + conflict modal; defer share sheet to M3.”
OUT-C2-3Deferred + reasonM2E3S1 deferred — SPIKE-1 open
Filled example: Sprint 14 commits M2E1S2, M2E1S3a; goal in Linear + WBS.md.
