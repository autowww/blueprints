Agentic SDLC

Humans own intent and acceptance; agents and CI amplify execution under policy. Not a replacement for Scrum/Kanban/XP/phased—an execution layer. Canonical: agentic-sdlc.md.

Principles

  1. Humans own intent — goals, acceptance, ethics, release.
  2. Agents amplify execution — code, tests, drafts—under review, CI, security.
  3. Same lifecycle — A–F still apply; throughput changes, not the need for specs/traceability where required.
  4. Transparency — PRs and CI are inspectable; chat is not the system of record.

Flow (diagrams)

flowchart LR H[Human intent / AC] --> A[Agents + IDE AI] A --> R[Human review] R --> CI[CI / gates] CI --> REPO[Repo + work-unit ids] REPO --> H

Loop implies planning and retro still close the cycle—see Ceremonies.

Risks

  • Review bottleneck — smaller PRs, pairing, WIP limits on “in review.”
  • Metric gaming — prefer work-unit completion over raw commit counts.
  • Compliance — sign-offs and evidence outside chat; see project TRACKING-CHALLENGES.md.
