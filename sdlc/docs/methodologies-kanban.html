Kanban

Flow, WIP limits, policies — and why cycle time usually comes from the board, not git.

Process overview

  • Visualize work on a board; limit WIP to expose bottlenecks and reduce multitasking.
  • Pull: next item enters “Doing” only when there is capacity — policies (Definition of Ready/Done) should be explicit.
  • Metrics: lead time and cycle time come from board history; git shows activity, not queue time.
