Stage gates

Explicit go / kill / pivot decision points between PDLC phases — preventing wasted investment by requiring evidence before proceeding.

Gates G1–G5

GateBetweenKey questionEvidence required
G1P1 → P2Is the problem worth solving?Research synthesis, opportunity size, strategic fit
G2P2 → P3Does a viable solution exist?Validation evidence (desirability + feasibility + viability)
G3P3 → SDLCShould we invest in building this?Business case, funded resources, defined success metrics
G4SDLC → P4Is it ready to launch?Shippable increment, GTM plan, support readiness
G5P5 → P6Should we continue investing?Outcome metrics vs targets, competitive position, strategic fit

Gate decisions

DecisionMeaningAction
GoEvidence meets criteriaFund next stage, assign resources
KillEvidence shows initiative should not proceedStop work, reallocate, document learnings
PivotDifferent direction more promisingRecycle to earlier stage with revised framing
HoldInsufficient evidence to decideTime-boxed investigation, then re-gate

Definition of Done (product initiative)

A product initiative is done when:

  1. The target outcome (P3 success metrics) is achieved, explicitly accepted, or the initiative is pivoted/cancelled with documented reasoning.
  2. The experiment log captures validation evidence from P1–P2 and iteration evidence from P5.
  3. The product vision and metrics dashboard reflect current state.
  4. SDLC delivery obligations are met per SDLC.md §3 (story DoD) and §4 (epic DoD).
  5. If sunset: migration, communications, and data handling are complete per the sunset plan.
