PDLC ↔ SDLC bridge

How PDLC and SDLC relate — SDLC sits inside PDLC as the Build & Release engine. Full detail: PDLC-SDLC-BRIDGE.md.

The core distinction

DimensionSDLCPDLC
Core questionHow do we build this correctly?Should we build this at all?
ScopeRequirements → deployProblem validation → sunset
Primary ownerEngineering / deliveryProduct manager / product trio
Success metricVelocity, defect rate, DORAAdoption, retention, NPS, revenue
Risk focusTechnical (bugs, perf, security)Market (desirability, viability)
Failure modeShip late, ship buggyShip the wrong thing
End stateDeployed to productionProduct retired or repositioned

Phase alignment

PDLC phases P1–P3 are upstream of SDLC (validate before building). SDLC phases A–F handle delivery. PDLC phases P4–P6 are downstream (launch, grow, sunset after building).

PDLC PhaseSDLC PhaseRelationship
P1 Discover Problem—Upstream: no engineering commitment yet
P2 Validate Solution—Upstream: prototypes, not production code
P3 Strategize—Upstream: outputs become Phase A inputs
—A–FSDLC delivery: build validated ideas
P4 LaunchF (partial overlap)Release = deployable; Launch = market activation
P5 GrowA (continuous loop)Growth insights feed next discovery cycle
P6 Mature / Sunset—Downstream: strategic lifecycle decisions

Anti-patterns

Anti-patternDescriptionFix
Build trapSDLC without PDLC discovery — shipping features nobody usesAdd P1–P2 before committing engineering
Research trapInfinite discovery, never buildingTime-box P1–P2; set explicit gate criteria
Launch and forgetNo P5 measurement after shippingDefine success metrics before build; staff P5
Metric theaterMeasuring outputs instead of outcomesSeparate SDLC metrics from PDLC metrics
