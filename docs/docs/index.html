Documentation blueprint

Templates and bootstrap tooling for the entire docs/ tree — product-functional prose, requirements & WBS, architecture, ADRs, development, CI/CD, testing, release, security, and operations.

Source of truth: Markdown in blueprints/docs/ (STRUCTURE.md, README.md). When those change, update this HTML and MAINTENANCE.md.

Overview

This blueprint provides ready-to-use templates for every area of a project's docs/ tree. It complements blueprints/sdlc/ (process methodology) and blueprints/ide/ (IDE agent instructions). Run the init script once and get a complete documentation scaffold.

Mutable content lives under docs/ in the consuming repo after bootstrap — never inside blueprints/docs/.

Principles

PrincipleMeaning
Audience-firstReadable without the backlog tool.
Just enoughAdd docs when they reduce ambiguity, onboarding cost, or operational risk.
Single source of truthOne canonical place per topic; link to requirement IDs.
Docs as codeDocumentation lives in the repository; changes use the same review practices as code.
TraceabilityStable IDs (M1E3S2, R1, etc.) link planning ↔ specs ↔ tests.

Full docs/ layout

The init script creates this structure:

docs/
  INDEX.md                        Hub: links to every documentation area
  PROJECT.md                      Stack, compliance, repo context
  ROADMAP.md                      Milestone + epic status + % complete

  product/                        Product functional — vision, journeys, features
    vision/ · personas/ · capabilities/ · journeys/ · features/ · data/ · integrations/

  requirements/                   Spec-driven breakdown (WBS, risks, traceability)
    INDEX.md · STRUCTURE-PROPOSAL.md · WBS.csv / WBS.md
    traceability/                 themes-matrix.csv · tests-matrix.csv
    risks/                        README.md · RBS.md · register.csv · items/
    milestones/M1/epics/…         Nested: milestone → epic → story → task

  architecture/                   System overview, diagrams
  adr/                            Architecture decision records
  development/                    Build, style, CI/CD, quality gates
  testing/                        Scope-level test plans
  release/                        Distribution, signing, compliance
  security/                       Non-secret security notes
  operations/                     Runbooks, ops procedures

Bootstrap

From the repository root:

./blueprints/docs/scripts/init-docs-workspace.sh "Project Name"

This copies all templates into docs/, strips .template. from filenames, replaces {{PROJECT_NAME}} and {{DATE}}. Options:

  • --force — overwrite existing files (default: skip if exists)
  • -h — show usage

docs/ROADMAP.md is seeded from blueprints/sdlc/templates/ROADMAP.template.md.
