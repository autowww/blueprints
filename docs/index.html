Blueprints

Reusable documentation and process packages for software projects — organized into lifecycles, disciplines, and support packages.

How it fits together

Three lifecycle questions drive the framework. PDLC asks “are we building the right product?” — discovery through sunset. SDLC asks “are we building the product right?” — phases A–F from discover to release. PM governs delivery: schedule, budget, risk. BA provides the analytical techniques both lifecycles rely on. Product Docs defines where product-functional prose lives.

PDLC — Product Development Lifecycle P1 Discover → P2 Validate → P3 Strategize → P4 Launch → P5 Grow → P6 Sunset
SDLC — Software Delivery Lifecycle A Discover → B Specify → C Design → D Build → E Verify → F Release

Adopt in your repo

1. Add as a git submodule under blueprints/:

git submodule add https://github.com/autowww/blueprints.git blueprints

2. Bootstrap project-specific folders (sdlc/, docs/product/) using the templates in each package.

3. Open blueprints/docs/index.html in a browser for this portal, or serve with python -m http.server.
