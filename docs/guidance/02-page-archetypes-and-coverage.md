# Page archetypes and required coverage

Use these archetypes so pages do not all collapse into the same generic template.

## 1. Overview page

Use for: product overview, capability overview, concept overview.

Required coverage:

- what it is
- why it exists
- who it is for
- when to use it / when not to use it
- capability summary
- relationship to nearby products or concepts
- one primary mental-model visual
- role/action matrix
- what to read next

Recommended structured elements:

- capability table
- role matrix
- comparison table
- glossary

Recommended visuals:

- context diagram
- relationship tree
- bento grid of capabilities

## 2. Quickstart page

Use for: first hour, install and run, first session.

Required coverage:

- outcome of the quickstart
- prerequisites
- exact steps
- verification after each major step
- known pitfalls
- links to deeper guides

Recommended structured elements:

- prerequisite table
- step/verify table
- common mistakes callout

Recommended visuals:

- linear flow diagram
- checklist visual
- split layout with screenshot on one side and step notes on the other

## 3. Tutorial 101 page

Use for: first successful use.

Required coverage:

- what the user will accomplish
- why that first workflow matters
- step-by-step happy path
- expected screen or output states
- how to verify success
- one worked example
- next actions after success

Recommended structured elements:

- path table
- state checklist
- example table

Recommended visuals:

- flow diagram
- screenshot or UI callout map
- before/after state summary

## 4. Tutorial 201 page

Use for: normal recurring usage.

Required coverage:

- recurring jobs to be done
- decision points or operating modes
- examples from real use
- shortcuts / judgment guidance
- anti-patterns or common mistakes

Recommended structured elements:

- decision matrix
- job-to-be-done table
- do / avoid table

Recommended visuals:

- swimlane or decision flow
- board/columns visual
- comparison matrix

## 5. Tutorial 301 page

Use for: advanced or power-user usage.

Required coverage:

- advanced scenarios
- limits and trade-offs
- configuration or choices that affect user outcomes
- interoperability with adjacent workflows
- recovery / rollback / fallback behaviors

Recommended structured elements:

- option matrix
- edge-case table
- capability/trade-off table

Recommended visuals:

- decision tree
- state diagram
- comparison/status visual

## 6. Adoption / rollout page

Use for: adoption paths, team rollout, rollout stages.

Required coverage:

- audience or team segments
- decision factors
- recommended path by segment
- rollout phases
- success criteria
- risks and mitigations

Recommended structured elements:

- path matrix
- phase table
- risk matrix
- stakeholder map

Recommended visuals:

- decision tree
- roadmap / timeline
- org or swimlane diagram

## 7. Troubleshooting page

Use for: symptoms and fixes.

Required coverage:

- symptom-first entry
- likely causes
- fix steps in order of cheapest/most probable first
- when to escalate or switch guides
- what evidence to gather

Recommended structured elements:

- symptom/cause/action table
- triage checklist
- escalation matrix

Recommended visuals:

- decision flow
- state/status table
- warning callouts

## 8. Reference-lite page

Use for: public reference that supports use, not internals.

Required coverage:

- concise explanation of terms or options
- what each item means to a user
- default expectations
- examples

Recommended structured elements:

- glossary table
- options table
- examples matrix

Recommended visuals:

- small taxonomy tree
- matrix

## Standard section order by archetype

Not every page needs identical headings. Use the archetype to choose the right shape.

### Overview

- Summary
- Why it exists
- How it fits
- Core concepts
- Main workflows / capabilities
- Who should use what
- Common pitfalls
- Related topics

### Quickstart / Tutorial

- Summary
- Outcome
- Prerequisites
- Steps
- Verify success
- Variations / branches
- Common issues
- Next topics

### Adoption / Rollout

- Summary
- Decision factors
- Path options
- Recommended path by audience
- Rollout plan
- Risks and mitigations
- Success measures
- Next topics

### Troubleshooting

- Summary
- Fast triage
- Symptom index
- Common causes
- Fixes
- When to escalate
- Related topics
